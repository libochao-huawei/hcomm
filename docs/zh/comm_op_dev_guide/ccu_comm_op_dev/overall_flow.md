# 总体流程

HCCL通信算子的开发流程如下所示：

![](figures/ccu_dev_flow.png)

1. **定义算子接口**：依据通信算子的功能作用，定义通信算子接口。
2. **查询拓扑信息**：获取通信域的网络层级、节点数量等信息。
3. **算法选择（可选）**：为最大化通信性能，往往在不同拓扑和数据量下采用不同算法实现，因此需要基于拓扑信息，选择最优的通信算法。
4. **创建资源**：申请通信算法所需要的内存、Thread、Notify、Channel等资源，完成CCU kernel的注册和资源申请。
5. **算子下发**：下发算子Kernel，交给通信引擎执行。
6. **算法执行**：利用通信资源，通过CCU进行各个rank进行同步和数据搬运，完成集合通信操作。
